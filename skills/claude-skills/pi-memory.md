# 🧠 Pi Memory Tool 🦞

Save durable information to persistent memory that survives across sessions. I'll use this proactively during our conversation.

## 🎯 Target Stores
- **`user`**: Who the user is — name, role, preferences, communication style, pet peeves.
- **`memory`**: My notes — environment facts, project conventions, tool quirks, and technical lessons learned.
- **`journal`**: Session-specific insights, decisions, and outcomes (Journal-like logging).

## 🕒 When to Capture (Be Proactive!)
- **Corrections**: User corrects me or says "remember this" / "don't do that again."
- **Preferences**: User shares a preference, habit, or personal detail (name, coding style, leadership style).
- **Environment**: I discover something stable about the environment (OS, installed tools, project structure).
- **Lessons**: I learn a convention, API quirk, or workflow specific to this user's setup.
- **Insights**: I identify a stable fact that will be useful again in future sessions.

## 📜 How to Use
- **add**: Append a new entry.
- **replace**: Find an entry containing a substring and update it.
- **remove**: Delete an entry by its content.

## ♊ Best Practices
- **Minimalistic**: Keep entries compact and focused on facts that still matter later.
- **No Task Progress**: Do NOT save temporary task progress or completed work logs. Use the journal for session outcomes instead.
- **Priority**: User preferences and corrections > environment facts > procedural knowledge.
